CSV Import

Upload CSV files and import inventory in 3 steps: Upload → Preview → Import. (This page stages CSV for preview.)

CSV Upload Admin Tool
Import policy
  • Assets are upserted by external_key.
  • Software preview validates that the target asset already exists for external_key.
  • source is the import channel and defaults to CSV_UPLOAD when omitted.
  • source_type is taken from the CSV row when provided; otherwise it defaults to CSV_UPLOAD.
  • CSV import uses header names directly. Column names must match the expected headers.
  • Software key for append/upsert is asset + vendor + product + version (DB UNIQUE).
  • For full host snapshots, use Replace asset software in the Software CSV Preview page.

Upload Assets CSV

Required headers: external_key, name
Common supported headers include asset_type, owner, note, platform, os_version, system_uuid, serial_number, hardware_vendor, hardware_model, computer_name, local_hostname, cpu_brand, cpu_physical_cores, cpu_logical_cores, arch, os_name, os_build, os_major, os_minor, os_patch, and last_seen_at.

Upload Software CSV

Required headers: external_key, product (version is optional)
Common supported headers include vendor, version, install_location, installed_at, package_identifier, arch, source_type, last_seen_at, type, source, vendor_raw, product_raw, version_raw, publisher, bundle_id, package_manager, install_source, edition, channel, release, and purl.
Note: assets must already exist (import assets first). CSV import uses exact header names rather than JSON-style alias mapping.
For full inventory snapshots, choose Replace asset software in the preview step. Use append/upsert only when you are intentionally adding incremental rows.
