CVE Delta Update (API)

Fetch recently modified CVEs via the NVD API (lastModified window), upsert vulnerabilities, evaluate software matches, and create or update alerts.

For bulk ingestion from CVE JSON 2.0 feeds, use “Update Vulnerability Data”.

Run delta update

Typical prerequisites
  • Import your inventory (assets / software) via Import Inventory (CSV).
  • Update the product dictionary and review product mapping for higher match quality: Update Product Dictionary → Review Product Mapping.
Matching works via canonical IDs / normalized vendor+product first, and also supports exact cpe_name matching as a fallback. Version range checks may become UNCONFIRMED when software version is missing or cannot be parsed.
Fetches CVEs whose lastModified is within (now - daysBack) … now. Smaller values = faster; larger values = more catch-up.
Limits how many CVE items are fetched from the API in one run. If the window contains more items than this cap, rerun with a smaller window or run multiple times.

Result

Interpretation: Fetch/Upsert affects the vulnerability store; Pairs/Alerts reflects the matching pass against your current software inventory.

Metric Value Description
fetched 0 Number of CVE items returned by the NVD API in this run.
vulnerabilitiesUpserted 0 Number of vulnerability rows saved or updated.
affectedCpesInserted 0 Number of affected product specifications inserted (includes version range metadata).
pairsFound 0 Total software × vulnerability candidate pairs evaluated as non-NO_MATCH.
alertsInserted 0 New alerts created (first time a pair becomes visible).
alertsTouched 0 Existing alerts updated (e.g., refreshed lastSeenAt, certainty or matchedBy updated).
Tip: If you only want to run the matching phase (no CVE fetch), use Generate Alerts.
