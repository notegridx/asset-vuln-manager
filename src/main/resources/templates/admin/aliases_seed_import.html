How to prepare Seed JSON

Import alias seeds for the CPE dictionary. Paste a JSON document that maps common vendor/product names to canonical CPE entries.

Canonical names must match the CPE dictionary name_norm values.

Example canonical names

Vendor:
microsoft

Product:
windows_server

Example JSON

{
  "vendors": [
    {
      "canonicalVendor": "microsoft",
      "aliases": [
        "microsoft corporation",
        "microsoft corp",
        "ms"
      ]
    }
  ],
  "products": [
    {
      "canonicalVendor": "microsoft",
      "canonicalProduct": "windows_server",
      "aliases": [
        "windows server",
        "microsoft windows server"
      ]
    }
  ]
}

Import rules

  • Aliases are normalized automatically
  • Existing aliases are skipped
  • Canonical entries must exist in the CPE dictionary
  • Conflicting aliases are ignored
  • Existing records are not overwritten

Typical workflow

  1. Identify unresolved vendor/product names
  2. Prepare alias seed JSON
  3. Import here
  4. Run linking again

Seed JSON files may be distributed separately.
