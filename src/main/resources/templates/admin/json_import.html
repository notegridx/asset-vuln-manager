JSON Import

Upload JSON files and import inventory in 3 steps: Upload → Preview → Import. Staging tables are used to keep Preview/Import separated.

JSON Upload Admin Tool
Import policy
  • Assets are upserted by externalKey.
  • Software preview validates that the target asset already exists for externalKey.
  • source is the import channel and defaults to JSON_UPLOAD when omitted.
  • source_type is taken from the payload for software rows when provided; otherwise it defaults to JSON_UPLOAD.
  • Software import supports both camelCase and snake_case for common osquery-style fields.
  • Software key for append/upsert is asset + vendor + product + version (DB UNIQUE).
  • For full host snapshots, use Replace asset software in the Software Preview page.

Upload Assets JSON

Required: externalKey, name
Accepts both camelCase and common snake_case keys such as asset_type, os_version, system_uuid, serial_number, hardware_vendor, hardware_model, computer_name, local_hostname, cpu_brand, cpu_physical_cores, cpu_logical_cores, os_name, os_build, os_major, os_minor, os_patch, and last_seen_at.

Upload Software JSON

Required: externalKey, product (version is optional)
Accepts camelCase and common snake_case osquery-style keys such as install_location, installed_at, package_identifier, source_type, vendor_raw, product_raw, version_raw, bundle_id, package_manager, install_source, and last_seen_at.
Note: assets must already exist (import assets first).
